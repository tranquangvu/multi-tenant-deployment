## Establish Rollback Strategy per Tenant

Implement rollback and recovery for application and database layers independently for each tenant.

## Deliverables:

- Rollback scripts or previous version restore logic
- Version tagging per tenant deployment
- DB rollback (Flyway undo or snapshot restore)

## Acceptance Criteria:

- Rollback can be executed per tenant
- Pipeline logs reflect rollback actions
- Rollback tested on at least one tenant successfully
